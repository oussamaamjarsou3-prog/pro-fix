## 5. CSS Cleanup Opportunities

### 5.1 `css/responsive.css` (2,128 lines) — Major Concern

| Issue | Evidence | Impact |
|-------|----------|--------|
| **Monolithic file** | Contains rules for home page (`categories`, `articles`, `newsletter`, `featured-car`), compare page (`compare-controls`, `winner-card`), car page, and generic utilities | High risk of unintended cascade |
| **Heavy `!important` usage** | Lines 153-169: `display:block !important`, `grid-template-columns:repeat(2, 1fr) !important` | Makes debugging and overriding difficult |
| **Home-page selectors in car context** | `.categories`, `.articles`, `.testimonials`, `.newsletter` are defined but never appear on car pages | Dead CSS on car pages, wastes bytes |
| **Redundant overflow guards** | Lines 27-45: explicit `overflow-x:hidden` on 12 selectors, then repeated on body at line 16 | One guard on `body` and `main` is sufficient |
| **No car-page-specific mobile breakpoints** | `@media (max-width:768px)` covers hero, section titles, footer — but no dedicated `.car-nav`, `.quick-info`, `.specs-table` mobile rules | Tables and sticky nav may break on narrow screens |

### 5.2 `css/car-page.css` (2,735 lines) — Moderate Concern

| Issue | Evidence | Impact |
|-------|----------|--------|
| **Single file, no modules** | All car page styles in one file: header, hero, nav, sections, calculator, gallery, footer | Hard to maintain; no dead-code elimination possible |
| **Hardcoded color values** | `rgba(59, 130, 246, ...)` (blue-500) repeated 40+ times instead of CSS variable | Theme changes require find-and-replace |
| **Magic numbers** | `border-radius: 99px` (pills), `z-index: 9999`, `z-index: 10000` | No z-index scale documented |
| **Redundant backdrop-filter** | Applied to `.info-card`, `.calc-result-card`, `.calc-total`, `.country-banner`, header | Performance cost on low-end mobile; should be optional |
| **Animation on all cards** | `.info-card:hover` has `transform: translateY(-6px) scale(1.02)` | `will-change` not declared; may cause jank on mobile |
| **No `prefers-reduced-motion`** | All transitions and animations unconditional | Accessibility violation for vestibular disorders |
| **Container query opportunity** | All grids use `@media` breakpoints; `container-type: inline-size` would let sections adapt independently | Section-level responsiveness is more robust than page-level |

### 5.3 `css/variables.css` (294 lines) — Minor Concern

| Issue | Evidence |
|-------|----------|
| **Utility classes mixed with variables** | `.skip-link`, `.search-results`, `.sr-only` defined in `variables.css` instead of utility file |
| **Fallback values everywhere** | `var(--text-primary, #fff)` — fallback should be centralized, not per-declaration |

## 12. Conclusion

The `car-template.html` is a **feature-rich, visually polished template** with strong multilingual support and a solid section hierarchy. It successfully renders the RS7 as a comprehensive car page. However, before it can serve as the definitive master template for all future cars, **three categories of work** must be addressed:

1. **Parameterization:** The hardcoded RS7 path, language, and country must become generator-injected variables. This is the only true blocker.
2. **Accessibility:** The `hidden` → visible section transition needs `aria-live` support, and the mega-menu focus trap must be implemented.
3. **CSS architecture:** The monolithic `responsive.css` and `car-page.css` files need modularization, `prefers-reduced-motion` support, and reduction of dead code.

Once these are resolved, the template can be frozen and used to generate all 6 existing car pages plus future models without per-page drift.
