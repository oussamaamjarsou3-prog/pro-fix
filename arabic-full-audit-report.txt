ARABIC LOCALIZATION FULL AUDIT REPORT
=======================================
Generated after quick check + per-car review.

EXECUTIVE SUMMARY
-----------------
The pageText.ar sections and the ar.json locale are now 100% free of Latin characters,
but the generated Arabic car pages still show Spanish/English text because:

1. The renderer (car-renderer-registry.js, car-renderer.js) contains many hardcoded
   English/Spanish labels and fallback label maps that are not translated to Arabic.
2. The car data JSONs are missing Arabic translations for whole sections (tires,
   countryPricing, ownerReviews, profileData, usedGuide) and for some Arabic-specific
   fields (drivetrainAr, badgeAr, versions.nameAr, comparisons.nameAr/fullNameAr).
3. The ar.json "features" map is missing 43 keys used by version/comparison/options cards.
4. The country-manager.js fiscal table shows region names and currencies in English/Spanish
   because countryData does not provide Arabic region names and formatLocalMoney is hardcoded
   to Spanish formatting.
5. The renderers do not stringify objects safely, so some values appear as "[object Object]".

FISCAL TABLE ISSUES (system/js/country-manager.js)
-------------------------------------------------
- updateFiscalTable reads c.fiscalRows and prints region names directly.
- Region names (California, Texas, Florida, New York, Illinois, etc.) are not translated.
- formatLocalMoney uses Spanish locale and EUR/USD symbols; for Arabic it should use
  Arabic-Indic digits and Arabic formatting or at least the Arabic currency name.
- The table headers do translate via t("fiscal.region"), etc.

[object Object] BUGS
--------------------
- car-renderer-registry.js safety driverAssist: if a.driverAssist item is an object
  and a.name is itself an object, it is returned as-is and later rendered as [object Object].
- car-renderer-registry.js tires: if t.standard / t.recommended / t.optional items are
  objects, they are concatenated as [object Object].
- car-renderer-registry.js options: if opt.name is an object, it renders as [object Object].
- car-renderer.js versions: featLabel may be an object if t() returns an object.

RECOMMENDED FIX ORDER
---------------------
1. Add Arabic translations for the 43 missing feature keys in ar.json.

   by introducing a label map per language or using t() keys.
3. Add missing Arabic car data sections (tires, countryPricing, ownerReviews, profileData,
   usedGuide, safety, versions.nameAr, comparisons.nameAr/fullNameAr, drivetrainAr, badgeAr).
4. Fix [object Object] by coercing all displayed values to string with safe fallback.
5. Translate fiscal table region names and currency formatting for Arabic in country-manager.js.
